# ✍️ คู่มือการเขียนบทความและคอร์สเรียน (Content Creation Guide)

คู่มือฉบับนี้จะอธิบายวิธีการสร้างบทความใหม่ สร้างคอร์สเรียน และการแทรกรูปภาพประกอบในไฟล์ `.mdx` สำหรับเว็บไซต์ KruTeekidCode

---

## 📂 1. โครงสร้างการจัดเก็บไฟล์

ไฟล์เนื้อหาทั้งหมดจะถูกจัดเก็บในรูปแบบ **MDX** (Markdown + JSX) ซึ่งรองรับการเขียนข้อความปกติและการแทรกโค้ด หรือ Component พิเศษได้

โฟลเดอร์สำหรับเก็บไฟล์เนื้อหา:
- **บทความ (Blog):** `content/blog/`
- **คอร์สเรียน (Courses):** `content/courses/`

โฟลเดอร์สำหรับเก็บรูปภาพ:
- รูปภาพทั้งหมดที่ใช้ในเนื้อหา **ต้องนำไปเก็บไว้ในโฟลเดอร์** `public/images/` หรือสร้างโฟลเดอร์ย่อยด้านใน เช่น `public/images/blog/`

---

## 📝 2. การสร้างบทความใหม่ (Blog)

หากต้องการสร้างบทความใหม่ ให้ทำตามขั้นตอนดังนี้:

1. สร้างไฟล์ `.mdx` ใหม่ในโฟลเดอร์ `content/blog/` เช่น `my-new-article.mdx` (ตั้งชื่อไฟล์เป็นภาษาอังกฤษ พิมพ์เล็กทั้งหมด และใช้ขีดกลาง `-` แทนช่องว่าง)
2. ใส่ข้อมูลส่วนหัว (Frontmatter) ไว้บนสุดของไฟล์เสมอ เพื่อให้ระบบดึงข้อมูลไปแสดงผลได้อย่างถูกต้อง:

```mdx
---
title: "ชื่อบทความของคุณ"
description: "คำอธิบายสั้นๆ ของบทความ (จะแสดงบนการ์ดและ SEO)"
date: "2026-06-02"
tags: ["Technology", "Coding"]
coverImage: "/images/blog/cover-image.jpg"
---

เนื้อหาบทความของคุณเริ่มต้นที่นี่...
คุณสามารถเขียน Markdown ปกติได้เลย
```

---

## 🎓 3. การสร้างคอร์สเรียนใหม่ (Course)

วิธีการสร้างคอร์สจะคล้ายกับการสร้างบทความ:

1. สร้างไฟล์ `.mdx` ใหม่ในโฟลเดอร์ `content/courses/` เช่น `basic-react.mdx`
2. ใส่ข้อมูลส่วนหัว (Frontmatter) ดังนี้:

```mdx
---
title: "พื้นฐาน React สำหรับผู้เริ่มต้น"
description: "เรียนรู้การสร้างเว็บไซต์ด้วย React ตั้งแต่ศูนย์"
---

เนื้อหาคอร์สเรียน...
```

---

## 🖼️ 4. วิธีการใส่รูปภาพประกอบในเนื้อหา

ระบบของ Next.js จะดึงรูปภาพจากโฟลเดอร์ `public/` โดยอัตโนมัติ

**ขั้นตอนการใส่รูปภาพ:**
1. นำไฟล์รูปภาพของคุณ (เช่น `my-photo.jpg`) ไปวางไว้ในโฟลเดอร์ `public/images/`
2. ในไฟล์ `.mdx` ให้ใช้คำสั่ง Markdown รูปแบบนี้เพื่อแทรกรูปภาพ:

```markdown
![คำอธิบายรูปภาพสำหรับผู้พิการทางสายตา (Alt text)](/images/my-photo.jpg)
```

> ⚠️ **ข้อควรระวัง:** 
> - สังเกตว่า path ของรูปภาพจะเริ่มด้วย `/images/...` ทันที **ไม่ต้อง** ใส่คำว่า `public` ลงไปใน path
> - ชื่อไฟล์รูปภาพควรใช้ภาษาอังกฤษและตัวพิมพ์เล็ก เพื่อป้องกันปัญหาการอ่านไฟล์บนเซิร์ฟเวอร์ (Vercel)

**ตัวอย่างการเขียนจริง:**
```markdown
นี่คือหน้าตาของโปรแกรมที่เราจะสร้างกันในวันนี้:

![หน้าตาโปรแกรมเครื่องคิดเลข](/images/calculator-preview.png)

จากรูปด้านบนจะเห็นว่า...
```

## 💻 5. วิธีการแทรกโค้ด (Code Blocks)

หากคุณต้องการแสดงตัวอย่างโค้ดในบทความ ให้ใช้เครื่องหมาย Backtick (```) 3 ตัว คลุมโค้ดไว้ และระบุภาษาหลัง Backtick บรรทัดแรก:

<pre>
```javascript
function sayHello() {
  console.log("สวัสดีครับ!");
}
```
</pre>
