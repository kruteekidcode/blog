# ✍️ คู่มือการเขียนบทความและคอร์สเรียน (Content Creation Guide)

คู่มือฉบับนี้จะอธิบายวิธีการสร้างบทความใหม่ สร้างคอร์สเรียน และการแทรกรูปภาพประกอบในไฟล์ `.mdx` สำหรับเว็บไซต์ KruTeekidCode

---

## 📂 1. โครงสร้างการจัดเก็บไฟล์

ไฟล์เนื้อหาทั้งหมดจะถูกจัดเก็บในรูปแบบ **MDX** (Markdown + JSX) ซึ่งรองรับการเขียนข้อความปกติและการแทรกโค้ด หรือ Component พิเศษได้

โฟลเดอร์สำหรับเก็บไฟล์เนื้อหา:
- **บทความ (Blog):** `content/blog/`
- **คอร์สเรียน (Courses):** `content/courses/`

โฟลเดอร์สำหรับเก็บรูปภาพ:
- รูปภาพทั้งหมดที่ใช้ในเนื้อหา **ต้องนำไปเก็บไว้ในโฟลเดอร์** `public/images/` หรือสร้างโฟลเดอร์ย่อยด้านใน เช่น `public/images/blog/`

---

## 📝 2. การสร้างบทความใหม่ (Blog)

### วิธีที่ 1: ใช้สคริปต์ (แนะนำ — ง่ายที่สุด)

เปิด Terminal ในโฟลเดอร์โปรเจ็ค แล้วพิมพ์:

```bash
npm run new:post my-new-article "ชื่อบทความภาษาไทย"
```

ระบบจะสร้างไฟล์ `content/blog/my-new-article.mdx` พร้อม Frontmatter ครบทุกช่องให้อัตโนมัติ และตั้งสถานะเป็น **draft** (ฉบับร่าง) ให้ด้วย

### วิธีที่ 2: สร้างไฟล์เอง

1. สร้างไฟล์ `.mdx` ใหม่ในโฟลเดอร์ `content/blog/` เช่น `my-new-article.mdx` (ตั้งชื่อไฟล์เป็นภาษาอังกฤษ พิมพ์เล็กทั้งหมด และใช้ขีดกลาง `-` แทนช่องว่าง)
2. ใส่ข้อมูลส่วนหัว (Frontmatter) ไว้บนสุดของไฟล์เสมอ เพื่อให้ระบบดึงข้อมูลไปแสดงผลได้อย่างถูกต้อง:

```mdx
---
title: "ชื่อบทความของคุณ"
description: "คำอธิบายสั้นๆ ของบทความ (จะแสดงบนการ์ดและ SEO)"
date: "2026-06-02"
tags: ["Technology", "Coding"]
coverImage: "/images/blog/cover-image.jpg"
draft: true
---

เนื้อหาบทความของคุณเริ่มต้นที่นี่...
คุณสามารถเขียน Markdown ปกติได้เลย
```

### ระบบฉบับร่าง (Draft)

- ใส่ `draft: true` ใน Frontmatter = บทความจะ**ไม่แสดงบนเว็บจริง** แต่ยังดูตัวอย่างได้ตอนรัน `npm run dev` ที่เครื่องตัวเอง
- เมื่อเขียนเสร็จพร้อมเผยแพร่ ให้**ลบบรรทัด** `draft: true` ออก แล้ว push ขึ้น git — บทความจะขึ้นเว็บอัตโนมัติ
- **ข้อควรระวัง:** วันที่ (`date`) ต้องอยู่ในรูปแบบ `YYYY-MM-DD` เท่านั้น เช่น `"2026-06-11"` ถ้าพิมพ์ผิดระบบจะเตือนตอน build

## 🎓 3. การสร้างคอร์สเรียนใหม่ (Course)

วิธีการสร้างคอร์สจะคล้ายกับการสร้างบทความ:

1. สร้างไฟล์ `.mdx` ใหม่ในโฟลเดอร์ `content/courses/` เช่น `basic-react.mdx`
2. ใส่ข้อมูลส่วนหัว (Frontmatter) ดังนี้:

```mdx
---
title: "พื้นฐาน React สำหรับผู้เริ่มต้น"
description: "เรียนรู้การสร้างเว็บไซต์ด้วย React ตั้งแต่ศูนย์"
level: "เริ่มต้น"
lessons: 10
duration: "6 ชั่วโมง"
order: 3
---

เนื้อหาคอร์สเรียน...
```

**ความหมายของช่องเสริม (ใส่หรือไม่ใส่ก็ได้):**

| ช่อง | ความหมาย |
| :--- | :--- |
| `level` | ระดับความยาก เช่น "เริ่มต้น", "กลาง", "สูง" (แสดงเป็นป้ายบนการ์ด) |
| `lessons` | จำนวนบทเรียน (ตัวเลข) |
| `duration` | ระยะเวลาเรียนโดยประมาณ |
| `order` | ลำดับการแสดงผล (เลขน้อยแสดงก่อน) |

## 🖼️ 4. วิธีการใส่รูปภาพประกอบในเนื้อหา

ระบบของ Next.js จะดึงรูปภาพจากโฟลเดอร์ `public/` โดยอัตโนมัติ

**ขั้นตอนการใส่รูปภาพ:**
1. นำไฟล์รูปภาพของคุณ (เช่น `my-photo.jpg`) ไปวางไว้ในโฟลเดอร์ `public/images/`
2. ในไฟล์ `.mdx` ให้ใช้คำสั่ง Markdown รูปแบบนี้เพื่อแทรกรูปภาพ:

```markdown
![คำอธิบายรูปภาพสำหรับผู้พิการทางสายตา (Alt text)](/images/my-photo.jpg)
```

> ⚠️ **ข้อควรระวัง:** 
> - สังเกตว่า path ของรูปภาพจะเริ่มด้วย `/images/...` ทันที **ไม่ต้อง** ใส่คำว่า `public` ลงไปใน path
> - ชื่อไฟล์รูปภาพควรใช้ภาษาอังกฤษและตัวพิมพ์เล็ก เพื่อป้องกันปัญหาการอ่านไฟล์บนเซิร์ฟเวอร์ (Vercel)

**ตัวอย่างการเขียนจริง:**
```markdown
นี่คือหน้าตาของโปรแกรมที่เราจะสร้างกันในวันนี้:

![หน้าตาโปรแกรมเครื่องคิดเลข](/images/calculator-preview.png)

จากรูปด้านบนจะเห็นว่า...
```

## 💻 5. วิธีการแทรกโค้ด (Code Blocks)

หากคุณต้องการแสดงตัวอย่างโค้ดในบทความ ให้ใช้เครื่องหมาย Backtick (```) 3 ตัว คลุมโค้ดไว้ และระบุภาษาหลัง Backtick บรรทัดแรก:

<pre>
```javascript
function sayHello() {
  console.log("สวัสดีครับ!");
}
```
</pre>

## 🔗 6. การสร้างลิงก์เชื่อมโยงไปยังเว็บไซต์อื่นแบบเปิดแท็บใหม่
ใน Markdown ปกติ หากเราเขียนลิงก์แบบ `[ข้อความ](URL)` เมื่อผู้อ่านคลิกลิงก์ จะเป็นการเปิดลิงก์ในหน้าต่างเดิม ซึ่งทำให้ผู้อ่านออกจากเว็บไซต์ของเราไป

หากต้องการให้คลิกแล้ว **เปิดแท็บใหม่** (เทียบเท่ากับแท็ก HTML `<a target="_blank">`) ให้เขียนโค้ด HTML `<a>` ลงในไฟล์ `.mdx` ดังนี้:

```html
<a href="https://www.google.com" target="_blank" rel="noopener noreferrer">คลิกเพื่อไป Google (เปิดแท็บใหม่)</a>
```

> 💡 **ข้อมูลเพิ่มเติมเพื่อความปลอดภัย:**
> - `target="_blank"` คือคำสั่งสั่งให้เปิดหน้าต่าง/แท็บใหม่
> - `rel="noopener noreferrer"` เป็นมาตรฐานความปลอดภัยที่จำเป็นเพื่อป้องกันไม่ให้หน้าเว็บภายนอกเข้าถึงข้อมูลบางอย่างของหน้าเว็บเราได้

## 🎥 7. การแชร์วิดีโอ (YouTube & Vimeo)
คุณสามารถนำวิดีโอมาฝัง (Embed) เพื่อให้แสดงผลและเล่นวิดีโอภายในบทความของคุณได้ทันที โดยการนำโค้ด `<iframe>` มาวางลงในไฟล์ `.mdx` ได้เลยครับ

### 🔴 การฝังวิดีโอจาก YouTube:
1. เข้าไปที่วิดีโอ YouTube ที่ต้องการแชร์
2. คลิกปุ่ม **"แชร์" (Share)** ใต้วิดีโอ
3. เลือกเมนู **"ฝัง" (Embed)**
4. คัดลอก (Copy) โค้ด `<iframe>` ทั้งหมดที่ปรากฏ
5. นำโค้ดไปวางในไฟล์ `.mdx` ของคุณได้ทันที

**ตัวอย่างโค้ด YouTube:**
```html
<iframe 
  width="100%" 
  height="315" 
  src="https://www.youtube.com/embed/dQw4w9WgXcQ" 
  title="YouTube video player" 
  frameborder="0" 
  allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" 
  allowfullscreen
></iframe>
```

### 🔵 การฝังวิดีโอจาก Vimeo:
1. เข้าไปที่วิดีโอ Vimeo ที่ต้องการแชร์
2. คลิกไอคอน **"แชร์" (Share)** (รูปเครื่องบินกระดาษ)
3. ที่หัวข้อ **"ฝัง" (Embed)** ให้คัดลอกโค้ดทั้งหมด
4. นำโค้ดไปวางในไฟล์ `.mdx`

**ตัวอย่างโค้ด Vimeo:**
```html
<iframe 
  src="https://player.vimeo.com/video/76979871?h=8272103f6b" 
  width="100%" 
  height="360" 
  frameborder="0" 
  allow="autoplay; fullscreen; picture-in-picture" 
  allowfullscreen
></iframe>
```

## 🎵 8. การแชร์ไฟล์เสียง (SoundCloud & ไฟล์ MP3)
คุณสามารถใส่เครื่องเล่นเสียง (Audio Player) ในบทความได้ ทั้งจากบริการของ SoundCloud และเล่นไฟล์เสียงตรง (.mp3)

### 🟠 การฝังเครื่องเล่นเสียงจาก SoundCloud:
1. ไปที่เพลงหรือคลิปเสียงใน SoundCloud
2. คลิกปุ่ม **"แชร์" (Share)**
3. เลือกแท็บ **"ฝัง" (Embed)**
4. คัดลอกโค้ด `<iframe>` ในช่องข้อความ
5. นำมาวางในไฟล์ `.mdx`

**ตัวอย่างโค้ด SoundCloud:**
```html
<iframe 
  width="100%" 
  height="166" 
  scrolling="no" 
  frameborder="no" 
  allow="autoplay" 
  src="https://w.soundcloud.com/player/?url=https%3A//api.soundcloud.com/tracks/123456789&color=%23ff5500&auto_play=false&hide_related=false&show_comments=true&show_user=true&show_reposts=false&show_teaser=true"
></iframe>
```

### 🔈 การฝังไฟล์ MP3 โดยตรง:
หากต้องการให้เล่นไฟล์เสียง (.mp3) ที่อัปโหลดไว้ สามารถใช้แท็ก `<audio>` ของ HTML5 ได้ทันที:

- **กรณีเก็บไฟล์เสียงไว้ในโปรเจกต์:** นำไฟล์ไปวางไว้ที่โฟลเดอร์ `public/audio/` (หากไม่มีให้สร้างโฟลเดอร์ขึ้นมา) แล้วเขียนโค้ดดังนี้:
  ```html
  <audio controls src="/audio/my-podcast.mp3"></audio>
  ```
- **กรณีดึงไฟล์เสียงจากลิงก์ภายนอก:**
  ```html
  <audio controls src="https://example.com/podcast.mp3"></audio>
  ```
