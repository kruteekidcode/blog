## 📂 หน้าที่ของแต่ละ Directory และไฟล์หลัก

### 1. โฟลเดอร์ `app/` (Web Router & Layouts)
เป็นหัวใจหลักในการกำหนดโครงสร้างหน้าเพจของ Next.js 16 (App Router)
* **`app/layout.tsx`**: โครงร่างครอบเว็บไซต์ทั้งหมด ใส่ฟอนต์ **Inter** (ภาษาอังกฤษ) และ **Noto Sans Thai Looped** (ภาษาไทย) ผ่านระบบ `next/font` เพื่อความเร็วและความสวยงามสม่ำเสมอทุกเครื่อง พร้อมเรียกใช้แถบเมนูหัวเว็บ (`Navbar`) และท้ายเว็บ (`Footer`) และประกาศ RSS Feed
* **`app/globals.css`**: ดีไซน์ซิสเต็มของเว็บทั้งหมด โดยควบคุมสีผ่านตัวแปร CSS Variables ทั้งสไตล์หัวข้อ (`h1`-`h6`), ย่อหน้า (`p`), ลิงก์, ตาราง, และโค้ดโค้ดดิ้ง รวมถึงการควบคุม Responsive บนหน้าจอมือถือและแท็บเล็ต
* **`app/page.tsx`**: โค้ดหน้าแรกสุดของเว็บไซต์ ซึ่งจะดึงข้อมูลบทความล่าสุด 3 ชิ้น และวิชาเรียนล่าสุด 2 วิชามาจัดแสดงผลแบบอัตโนมัติ

### 2. โฟลเดอร์ `components/` (Reusable UI Components)
เก็บชิ้นส่วนขนาดเล็กของหน้าจอเพื่อให้โค้ดเป็นระเบียบและไม่ซ้ำซ้อน
* **`Navbar.tsx` & `Footer.tsx`**: แถบเมนูด้านบนและท้ายเว็บ โดย `Navbar` มีการเขียน JavaScript ในตัวเพื่อรองรับปุ่มแฮมเบอร์เกอร์สามขีดในหน้าจอมือถือ
* **`BlogCard.tsx` & `CourseCard.tsx`**: ออกแบบสไตล์ของการ์ดสรุปข้อมูลให้มีความสวยงาม มีปุ่มกดอ่าน และแสดงหมวดหมู่ (Tag)

### 3. โฟลเดอร์ `content/` (Content Database)
เปรียบเสมือนฐานข้อมูลของเว็บไซต์ โดยเก็บข้อมูลเป็นรูปแบบเอกสาร **MDX (Markdown + React Component)** ทำให้เขียนบทความได้ง่าย รวดเร็ว และไม่มีสไตล์โค้ดดิ้งมารบกวน
* ทุกครั้งที่คุณอยากเพิ่มบทความใหม่ ให้สร้างไฟล์ `.mdx` ลงในโฟลเดอร์ `content/blog/` ได้เลย
* ทุกครั้งที่คุณอยากเพิ่มหรือแก้ไขเนื้อหาหลักสูตร ให้สร้างหรือแก้ไขไฟล์ใน `content/courses/` ได้เลย

### 4. โฟลเดอร์ `lib/` (Backend Utilities)
* **`lib/content.ts`**: รวบรวมฟังก์ชันการทำงานส่วนลึก เช่น การสแกนไดเรกทอรีอ่านไฟล์บล็อก คัดแยกหัวข้อบล็อกตามวันที่ล่าสุด รวมถึงการใช้สูตรคำนวณระยะเวลาในการอ่านบทความ (เช่น `5 min`) และแปลงฟอร์แมตวันคริสต์ศักราชเป็นวันภาษาไทยปีพุทธศักราช (เช่น `2026-05-28` แปลงเป็น `28 พ.ค. 2569`)

### ตัวอย่างการแต่งบทความใน `content/blog/my-new-post.mdx`:
```markdown

date: "2026-05-31"
tags: ["เทคโนโลยี", "Coding"]
coverImage: ""
lang: "th"
draft: true
---

```

## 🚀 คำสั่งควบคุมโปรเจ็ค (CLI Commands Reference)

เปิดหน้า Terminal และเข้าโฟลเดอร์โปรเจ็ค เพื่อใช้งานคำสั่งต่อไปนี้:

* **สร้างบทความใหม่พร้อมโครง Frontmatter อัตโนมัติ** (สถานะเริ่มต้นเป็น draft):
  ```bash
  npm run new:post my-article-slug "ชื่อบทความภาษาไทย"
  ```

* **เปิดโปรเจ็คจำลองบนเครื่องเพื่อดูหน้าเว็บ** (โหมดนี้จะเห็นบทความ draft ด้วย):
  ```bash
  npm run dev
  ```
  *(เปิดเบราว์เซอร์ดูได้ที่: http://localhost:3000)*

* **ทดสอบระบบการ Build ก่อนนำขึ้นโฮสต์จริง** (บทความ draft จะถูกซ่อน):
  ```bash
  npm run build
  ```

* **เปิดรันโปรเจ็คในโหมด Production**:
  ```bash
  npm run start
  ```
